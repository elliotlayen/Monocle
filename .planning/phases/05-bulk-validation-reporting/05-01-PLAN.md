    **`bulk_scan_cmd` command:** Accepts `app: AppHandle`, `folder_path: String`, `file_pattern: String`, `operation_id: String`, `explorer_state: State<'_, ExplorerState>`. Stores a `CancellationToken` in `explorer_state.active_listings` keyed by `operation_id` (same pattern as `list_directory_cmd`). Runs in `tokio::task::spawn_blocking`. Two-phase approach inside the blocking closure:
    - Phase 1: Collect matching files using `WalkDir::new(&folder_path).into_iter().filter_map(|e| e.ok()).filter(|e| !e.file_type().is_dir()).filter(|e| e.file_name().to_str().map(|n| pattern.matches(n)).unwrap_or(false))`. Collect into `Vec<PathBuf>` to get `total_files` count.
    - Phase 2: Iterate collected paths. For each: check `token_clone.is_cancelled()` (break if true, set `cancelled = true` on summary). Read raw bytes with `std::fs::read`. Call `detect_and_decode(&raw_bytes)` then `validate_characters(...)` (same pipeline as `read_file_cmd`). Compute per-file error/warning counts, status, running totals. Build `ScanFileResult` with `relative_path` computed by stripping `folder_path` prefix from full path. Emit `app.emit("scan-progress", ScanProgressPayload { ... })`. Throttle events: track `last_emit_time` via `std::time::Instant`, only emit if >= 50ms since last emit OR if it is the last file (per RESEARCH Pitfall 4 on event flooding). Skip unreadable files with `continue` on read error.
    - On completion or cancellation: return `ScanSummary` with all results.
    - Cleanup: remove operation_id from `explorer_state.active_listings` after spawn_blocking completes (same cleanup pattern as `list_directory_cmd`).

    **`cancel_scan_cmd` command:** Identical to existing `cancel_directory_cmd` -- looks up `operation_id` in `explorer_state.active_listings`, calls `token.cancel()`. Can literally delegate to the same `active_listings` map since scans and directory listings use the same token store.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | bulk_scan_cmd file_pattern param | mitigate | Validate pattern with `glob::Pattern::new()` which rejects invalid patterns; pattern is matched against filename only (not full path) preventing directory traversal |
| T-05-02 | Tampering | bulk_scan_cmd folder_path param | accept | folder_path comes from tree node which was already loaded via list_directory_cmd; Tauri fs plugin scopes constrain accessible paths |
| T-05-03 | Denial of Service | bulk_scan_cmd on huge directory | mitigate | CancellationToken allows user to cancel; 50ms event throttling prevents frontend flooding; two-phase approach shows file count before validation begins |
| T-05-04 | Information Disclosure | scan progress events | accept | Events contain file paths and validation status only, not file content; events are app-local (not network-transmitted) |
| T-05-SC | Tampering | npm/cargo installs | mitigate | walkdir (BurntSushi/rust-lang ecosystem, transitive dep already in Cargo.lock) and glob (rust-lang org) are well-established; shadcn progress is Radix UI official. All tagged [ASSUMED] in research -- human verification checkpoint below. |
</threat_model>
