    Part B -- Extend tree-filter.ts:

    Modify filterTreeNodes signature to accept optional parameters for issues-only filtering. Add an optional options parameter: { showIssuesOnly?: boolean; validationCache?: Map<string, { problems: ValidationProblem[] }>; folderBadgeCache?: Map<string, ValidationStatus> }.

    When showIssuesOnly is true:
    - A file node passes if validationCache has an entry for its path with at least one problem.
    - A folder node passes if folderBadgeCache has an entry for its path with status "error" or "warning", OR if any of its children pass the filter.

    - When showIssuesOnly is false or omitted, existing behavior is unchanged.

    Import ValidationProblem and ValidationStatus types from "../types".
