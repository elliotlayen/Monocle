  <behavior>
    - formatXml("<root><child attr=\"val\">text</child></root>") returns indented multi-line output with 2-space indent
    - formatXml("<root/>") returns "<root/>" (self-closing preserved)
    - formatXml("<root><a>text</a></root>") keeps text content inline with its element (no newline between open tag, text, and close tag)
    - formatXml("") returns "" (empty input returns empty)
    - formatXml("<root><unclosed>") returns the original malformed string unchanged (graceful fallback)
    - formatXml handles XML with comments: "&lt;!-- comment -->" is preserved with proper indentation
    - formatXml handles CDATA sections: "&lt;![CDATA[data]]>" is preserved with proper indentation
    - formatXml handles processing instructions: "&lt;?xml version=\"1.0\"?>" is preserved
    - formatXml handles namespaced elements: "soap:Envelope" tag names preserved
    - formatXml with custom indent: formatXml(xml, "    ") uses 4-space indent
  </behavior>
  <action>
    Create xml-formatter.ts following the exact pattern of xml-parser.ts: no external imports, pure TypeScript utility, single exported function formatXml(xmlString: string, indentStr?: string): string with default 2-space indent.

    Implementation approach: Use DOMParser to parse the XML string. If parseerror is detected, return the original string unchanged. Otherwise, recursively walk the DOM tree (document.childNodes for top-level PIs, then documentElement) building an array of indented lines. Handle these node types:
    - Element (nodeType 1): open tag with attributes, recurse children, close tag. Self-closing if no children. Inline text if single text child.
    - Text (nodeType 3): trim whitespace, skip if empty after trim.
    - Comment (nodeType 8): emit as &lt;!-- text -->.
    - CDATA (nodeType 4): emit as &lt;![CDATA[text]]>.
    - ProcessingInstruction (nodeType 7): emit as &lt;?name value?>.

    Attribute serialization: iterate el.attributes, join as name="value".

    Create xml-formatter.test.ts with // @vitest-environment jsdom header. Import from vitest (describe, it, expect). Write tests for each behavior listed above. Follow the same test structure as xml-parser.test.ts: short focused it() blocks.
  </action>
