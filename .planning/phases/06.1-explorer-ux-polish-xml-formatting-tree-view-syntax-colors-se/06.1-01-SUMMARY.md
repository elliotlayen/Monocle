## Implementation Details

### Task 1: XML Formatter Utility

Created `formatXml(xmlString, indentStr?)` that parses XML via DOMParser and recursively serializes the DOM tree with indentation. Handles all XML node types:

- **Elements**: open/close tags with attributes, self-closing for empty elements
- **Text**: trimmed inline content, kept on same line as parent element for single-text children
- **Comments**: preserved as `<!-- text -->`
- **CDATA**: preserved as `<![CDATA[text]]>`
- **Processing Instructions**: preserved as `<?target value?>`
- **Namespaced elements**: tag names like `soap:Envelope` preserved

Malformed XML returns the original string unchanged (graceful fallback via parsererror detection).

13 unit tests cover all behaviors including custom indent, nested depth, multiple attributes, and mixed content.

### Task 2: XML Element Tag Syntax Colors

Replaced `text-foreground` on the tagName span in xml-tree-node.tsx with purple/violet oklch colors:
- Light mode: `text-[oklch(0.55_0.18_280)]`
- Dark mode: `dark:text-[oklch(0.72_0.14_280)]`

This completes the syntax color palette: element tags (purple, hue 280), attributes (blue, hue 250), values (green, hue 150), CDATA (yellow, hue 60).
