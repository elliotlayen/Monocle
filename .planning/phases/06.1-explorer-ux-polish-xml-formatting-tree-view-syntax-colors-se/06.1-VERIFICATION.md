### Gaps Summary

Four gaps block full goal achievement, all sharing a common root cause: commit `e21fef0` ("revise UX polish based on review feedback") explicitly reverted features after they were implemented.

**Root cause:** A post-implementation review led to deliberate removal of:
1. Formatted date display in tree nodes (considered "too busy")
2. Issues-only filter toggle from sidebar (considered "incorrectly added")
3. `dateFilterPreset` preset dropdown (replaced with a calendar date-range dialog)
4. `SidebarToggle` shared component usage in `explorer-shell.tsx` (replaced with inline button strip)

These were not bugs or failures in implementation — they were intentional design changes made after the original phase tasks completed. The commit message and diff confirm explicit developer intent.

**What this means for the phase goal:**
- "readable XML formatting" — ACHIEVED (format toggle works)
- "proper syntax colors" — ACHIEVED (purple element tags)
- "working date display" — NOT ACHIEVED (date labels removed from tree nodes)
- "working date sort" — ACHIEVED (sort comparator wired)
- "ability to filter the tree to files with validation issues" — NOT ACHIEVED (issues-only filter removed)

The phase goal as stated is not fully achieved. Three of six UX requirements (UX-02 date display, UX-03 preset filter, UX-04 issues-only) are blocked. UX-06 (shared sidebar toggle) is partial (schema-graph uses shared, explorer uses inline).
