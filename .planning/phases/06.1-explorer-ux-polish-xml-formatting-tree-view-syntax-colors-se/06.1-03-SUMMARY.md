## Implementation Details

### Task 1: XML Format Toggle

Added a WrapText icon button in file-content-header.tsx that toggles between raw and formatted XML in source view. Format state is managed as local component state in file-content-area.tsx (the parent that owns content rendering) rather than in file-content-header.tsx, since the parent passes content to XmlSourceView.

Key implementation:
- `isFormatted` state resets to false when `activeTabId` changes
- Formatted content computed via `formatXml()` from Plan 01's xml-formatter utility
- Button only renders when `viewMode === "source"` and `tab.isXml` is true
- Tooltip indicates "Format XML (line numbers refer to original)" when off, "Show raw XML" when on
- Active state shown with `bg-accent` class

### Task 2: Issues-Only Filter

Extended the tree-filter module with a `FilterTreeOptions` interface supporting `showIssuesOnly`, `validationCache`, and `folderBadgeCache` parameters. The filter logic:

1. File nodes pass if they have problems in validationCache, or if they have no cache entry (unscanned = pass through)
2. Folder nodes pass if folderBadgeCache has "error" or "warning", or if any children pass, or if no subtree validation data exists
3. Files scanned clean (empty problems array) are filtered out
4. When both caches are empty, all nodes pass through (no data = no filter)

Added `showIssuesOnly` boolean and `toggleIssuesOnly` action to the explorer store. Wired the filter into folder-tree.tsx and added an AlertTriangle icon toggle button in explorer-sidebar.tsx header.

15 tests cover all filter behaviors including passthrough for empty caches, badge-based folder filtering, and combined text+issues filtering.
