## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline padding, tree indent markers |
| sm | 8px | Tree node internal padding, compact element gaps |
| md | 12px | Sidebar section padding (p-3), nav bar padding |
| lg | 16px | Settings form field gaps, section spacing |
| xl | 24px | Settings section separators, major content padding |
| 2xl | 32px | Empty state vertical spacing |
| 3xl | 48px | Page-level spacing (unused in this phase) |

Exceptions: Tree indentation uses 16px (ml-4) per nesting level, matching the existing schema-browser-sidebar pattern. Sidebar width default is 280px (matching `schema-browser-sidebar.tsx`), resizable via drag handle, min 200px, max 480px.

## Interaction Contracts

### Tree Node Expand/Collapse

| State | Visual | Interaction |
|-------|--------|-------------|
| Collapsed | `ChevronRight` icon, no children visible, no count badge | Click row or chevron to expand |
| Loading | `Loader2` spinning icon replaces chevron, "Loading... {N}s" after 3s (D-23) | Cancel button appears after 3 seconds alongside spinner |
| Expanded | `ChevronDown` icon, children rendered below, count badge appears | Click row or chevron to collapse |
| Failed | `AlertTriangle` icon, row in `text-muted-foreground` with error tooltip | Click to retry (D-26). No automatic retries. |
| Unreachable source | Entire source row in `text-muted-foreground opacity-60`, `AlertTriangle` icon, tooltip: "Unreachable -- check VPN connection" (D-22) | Expand to retry |

### Loading Behavior (D-23, D-25)

1. User clicks expand chevron on a tree node
2. Chevron immediately replaced by `Loader2` spinner (16x16, `animate-spin`)
3. After 3 seconds: cancel button (`X` icon, ghost variant, h-5 w-5) appears to the right of the node name, elapsed time text appears: "Loading... 3s", "Loading... 4s", etc.
4. After 15 seconds: timeout. Node collapses back, toast notification: "Folder listing timed out after 15 seconds"
5. Cancel click: Rust task dropped, node collapses to unexpanded state, no error message

### Favorites (D-16 through D-19)

| State | Visual | Interaction |
|-------|--------|-------------|
| Not favorited | No star visible on row by default | Hover reveals outline `Star` icon (ghost, 14x14). Right-click context menu shows "Add to Favorites". |
| Favorited | Filled `Star` icon in amber-500 color, always visible | Click star to unfavorite. Right-click context menu shows "Remove from Favorites". |
| Favorites section | "Favorites" label (text-xs font-semibold text-muted-foreground uppercase tracking-wide) pinned at top of source's client list | Only visible when source has at least one favorite (D-19). Favorited clients appear here AND in normal alphabetical position below. |

### Filter Bar (D-20, D-21)

| State | Visual | Interaction |
|-------|--------|-------------|
| Empty | `Search` icon + "Filter clients..." placeholder, `h-8 text-sm` matching schema sidebar | No filtering applied, all loaded nodes visible |
| Active | Input has typed text, matching nodes visible, non-matching hidden | Filters against all visible (already-expanded) nodes: client names, date folder names, filenames. No new network fetches. |
| No matches | Tree area shows centered text: "No matches found" in `text-sm text-muted-foreground` | Clear filter to restore tree |

### Sort Toggle (D-11)

| State | Visual | Interaction |
|-------|--------|-------------|
| Newest first (default) | `ArrowUpDown` icon button in sidebar header, tooltip: "Sort: newest first" | Click toggles to oldest first |
| Oldest first | Same button, tooltip: "Sort: oldest first" | Click toggles back to newest first |

Sort preference is session-only, resets to newest-first on app restart.

### Settings: Folder Sources Section (D-01 through D-07)

| State | Visual | Interaction |
|-------|--------|-------------|
| No sources | Centered text: "No folder sources configured" + "Add Source" button below | Click "Add Source" to create first entry |
| Has sources | Vertical list of source rows, each with: drag handle, path input, label input, tag input, browse button, remove button | Drag handle for reorder (D-05). All fields editable inline (D-07). |
| Adding source | New empty row appended at bottom with focus on path field | Type path or click browse button to select folder |
| Source unreachable | Path input shows amber border, warning text below: "Path unreachable -- check VPN connection" (D-04) | Source still saves. Warning is informational only. |
| Remove source | Click `Trash2` icon button (destructive ghost variant) | Immediate removal, no confirmation dialog. Sources are easy to re-add. |

Source row layout (each row):
```
[GripVertical] [Label input (w-32)] [Path input (flex-1)] [Tag input (w-24)] [FolderSearch browse] [Trash2 remove]
```

### Sidebar Resize (D-15, Phase 1 D-07)

| State | Visual | Interaction |
|-------|--------|-------------|
| Default | 280px wide sidebar, 4px border-right acts as resize handle | Cursor changes to `col-resize` on hover over right edge |
| Resizing | Sidebar width follows mouse, min 200px, max 480px | Drag right edge. Width persisted via `AppSettings.explorer_sidebar_width`. |
| Collapsed | Sidebar hidden (`-translate-x-full`), toggle button visible at `top-4 left-4` | Click toggle to expand. Matches `SidebarToggle` pattern. |

## Tree Node Visual Specifications

### Source Root Node
```
[ChevronRight/Down] [FolderSync icon, 16x16] [Label text, text-sm font-semibold] [Tag badge, secondary variant, text-xs]
```
- Indentation: 0 (root level)
- Row height: 32px (py-1.5 + content)
- Tag badge: `Badge variant="secondary"` showing source tag (e.g., "Inbound")
- If unreachable: entire row at `opacity-60`, `AlertTriangle` replaces `FolderSync`

### Client Node
```
  [ChevronRight/Down] [Folder/FolderOpen icon, 16x16] [Client code, text-sm font-medium] [Count badge, text-xs text-muted-foreground]
```
- Indentation: 16px (ml-4, level 1)
- Row height: 28px (py-1 + content)
- Count badge: plain text count (not Badge component), only shown after first expand
- Star icon: visible on hover (outline) or always visible (filled, amber) if favorited

### Date Folder Node
```
    [ChevronRight/Down] [Folder/FolderOpen icon, 14x14] [Raw date, text-sm] [(Formatted date), text-xs text-muted-foreground]
```
- Indentation: 32px (ml-4, level 2)
- Row height: 28px (py-1 + content)
- Display format: `"20251223"` followed by `"(Dec 23, 2025)"` in muted text (D-03, BRWS-03)
- Count badge: child file count, same pattern as client node

### File Node
```
      [FileCode/FileText icon, 14x14] [Filename, text-sm]
```
- Indentation: 48px (ml-4, level 3)
- Row height: 28px (py-1 + content)
- No chevron (leaf node)
- XML files: `FileCode` icon. Non-XML files: `FileText` icon in `text-muted-foreground` (D-14)
- Click does nothing in Phase 2 (D-13). Cursor remains `default`, not `pointer`.
