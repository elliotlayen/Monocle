    **folder-sources-section.tsx (per D-01, D-05, D-06, D-07, D-08):**
    Create FolderSourcesSection component. Consume useExplorerStore with useShallow to extract folderSources, reorderSources, saveSources, loadSources. Call loadSources in a useEffect on mount to populate from settings.

    Use local state for the working copy of sources (initialized from folderSources). Update local state on field changes, sync to store + persist on save.

    Actually, simpler approach: keep sources in store, use reorderSources for drag, use direct mutations that call saveSources after each change. Each field edit updates a local working copy, with an auto-save debounce or explicit save.

    Preferred approach for inline editing: maintain a local sources state initialized from store.folderSources via useEffect. All edits mutate local state. A separate save effect persists when local state differs from store state (debounced 500ms). This avoids flooding IPC on every keystroke.

    DnD wrapper: DndContext with closestCenter collision detection and onDragEnd handler. SortableContext with items mapped to source IDs, verticalListSortingStrategy. Render FolderSourceRow for each source.

    handleDragEnd: extract active.id and over.id, find old/new indices in local sources, call arrayMove (from @dnd-kit/sortable), update local state.

    Add Source button: Plus icon, variant outline, text "Add Source". On click, create new FolderSource with id from crypto.randomUUID(), empty path/label/tag, empty favorites. Append to local sources list.

    Remove handler: filter source by id from local sources list.

    Update handler: merge partial fields into the matching source by id.

    Browse handler: call open() from @tauri-apps/plugin-dialog with { directory: true, multiple: false, title: "Select folder source" }. If result is non-null, update the source's path field.

    Section layout per convention: div.space-y-6.px-1 containing heading div (h3 "Folder Sources" + p description "Configure root folder paths to browse in the explorer."), then sources list, then Add Source button.

    When no sources configured, show centered text "No folder sources configured" with the Add Source button below (per copywriting contract).

    Persist changes: use useEffect with a debounce (500ms setTimeout). When local sources differ from store.folderSources, call reorderSources(localSources) then saveSources(). Clear timeout on cleanup. This ensures every edit (field change, reorder, add, remove) eventually persists.
