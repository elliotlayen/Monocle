    **use-explorer-sidebar.ts (per D-15, RESEARCH.md pitfall 4):**
    Create the hook exporting useExplorerSidebar. Accepts initialWidth (number). Returns { width, isDragging, handleMouseDown }.

    State: localWidth (useState, initialized from initialWidth), isDragging (useState, false).

    handleMouseDown: set isDragging true, capture startX from event.clientX and startWidth from localWidth.

    useEffect when isDragging is true: attach global mousemove and mouseup listeners. On mousemove: calculate newWidth = startWidth + (event.clientX - startX), clamp between 200 and 480 (per UI-SPEC spacing scale). Set localWidth. On mouseup: set isDragging false, call a provided onWidthChange callback with the final localWidth (this is when persistence fires -- NOT during drag, avoiding the IPC flood pitfall). Clean up listeners on effect cleanup.

    Actually, simpler approach: the hook manages drag state and reports width. The consumer (sidebar) reads store.sidebarWidth for initial and calls store.setSidebarWidth on mouseup. The hook takes onWidthCommit callback.

    Revised interface:
    - useExplorerSidebar(initialWidth: number, onWidthCommit: (width: number) => void)
    - Returns { width: number, isDragging: boolean, startDrag: (e: React.MouseEvent) => void }
    - During drag: updates local width state at 60fps (mouse events). Does NOT call onWidthCommit.
    - On mouseup: calls onWidthCommit(finalWidth) once. This triggers the settings persistence in the store.
