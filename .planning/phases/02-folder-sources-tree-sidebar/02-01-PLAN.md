<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend -> Rust IPC | User-supplied folder paths cross from untrusted input to filesystem access |
| Filesystem -> App | Directory listing results come from potentially untrusted network shares |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | list_directory_cmd path parameter | mitigate | Use std::path::PathBuf for path handling -- prevents path traversal. Tauri commands run with OS-level permissions only. |
| T-02-02 | DoS | list_directory_cmd on huge directory | mitigate | 15-second timeout via tokio::time::timeout wrapping spawn_blocking. CancellationToken checked per-entry enables user cancel. |
| T-02-03 | Information Disclosure | settings.json folder paths | accept | Folder paths are not secrets -- they are user-configured network share locations. No passwords stored. |
| T-02-SC | Tampering | npm installs (@dnd-kit/core, @dnd-kit/sortable) | mitigate | Both packages verified in RESEARCH.md Package Legitimacy Audit: 5+ years old, ~2M downloads/week, github.com/clauderic/dnd-kit. slopcheck [OK]. |
</threat_model>
