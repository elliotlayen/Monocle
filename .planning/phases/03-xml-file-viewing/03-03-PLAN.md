    Create `src/features/explorer/hooks/use-file-actions.ts` exporting `useFileActions` hook. Follow the `useExport.ts` pattern with `useCallback` wrappers and error handling.

    Install the clipboard-manager npm package first: `npm install @tauri-apps/plugin-clipboard-manager`.

    Import `writeText` from `@tauri-apps/plugin-clipboard-manager`. Import `openPath` from `@tauri-apps/plugin-opener`. Import `save` from `@tauri-apps/plugin-dialog`. Import `writeFile` from `@tauri-apps/plugin-fs`. Import `showToast` from `@/features/notifications/store`.

    The hook returns an object with four async action functions:

    `copyPath(filePath: string)`: Call `writeText(filePath)`. On success, call `showToast({ type: "success", title: "Path copied to clipboard", duration: 2000 })`. On error, call `showToast({ type: "error", title: "Failed to copy path", duration: 3000 })`.

    `copyContent(content: string)`: Call `writeText(content)`. On success, call `showToast({ type: "success", title: "Content copied to clipboard", duration: 2000 })`. On error, call `showToast({ type: "error", title: "Failed to copy content", duration: 3000 })`.

    `openExternal(filePath: string)`: Call `openPath(filePath)` (per D-18, uses OS default app). On error, show error toast. No success toast needed (the app opens externally).

    `saveCopy(fileName: string, content: string)`: Call `save({ defaultPath: fileName })` to open native save dialog (per D-19). If user selects a path (not null), call `writeFile(destPath, new TextEncoder().encode(content))`. On success, show toast "File saved to {destPath}". On error, show error toast. If user cancels dialog (path is null), do nothing.

    Each function is wrapped in `useCallback` with no external dependencies (all args passed directly). No loading state needed -- these are quick operations. The hook is stateless.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User action -> clipboard | User-initiated clipboard write via Tauri plugin |
| User action -> OS shell | openPath delegates to OS to choose application |
| User action -> filesystem write | Save dialog + writeFile for save copy |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Tampering | openPath file path | accept | Path originates from directory listing (Phase 2). User controls folder sources. openPath delegates to OS which applies its own security policies. |
| T-03-07 | Tampering | writeFile save copy | accept | User explicitly chooses save destination via native OS dialog. Content is the file already read. No untrusted path injection. |
| T-03-08 | Information Disclosure | clipboard write | accept | User explicitly clicks "Copy". Content is from files they already have access to. Desktop app context. |
| T-03-SC | Tampering | npm install @tauri-apps/plugin-clipboard-manager | mitigate | Package passed slopcheck [OK] in RESEARCH.md audit. Official Tauri plugins-workspace. |
