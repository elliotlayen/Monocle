# Phase 3 -- UI Design Contract

> Visual and interaction contract for XML File Viewing. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | new-york (neutral base) |
| Component library | Radix UI (via shadcn) |
| Icon library | Lucide |
| Font | Inter (body), JetBrains Mono (branding/mono), system monospace (Monaco editor) |

Source: `components.json`, Phase 1 and Phase 2 UI-SPEC contracts.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, gap between tab close button and label |
| sm | 8px | Tab internal padding, header bar element gaps, tree node indent padding |
| lg | 16px | XML tree indentation per level (ml-4), content area padding, tab bar horizontal padding (`px-4`), content header bar padding (`px-4`), parse error banner padding (`px-4`) |
| xl | 24px | Section spacing between header bar and content body |
| 2xl | 32px | Empty state vertical spacing |
| 3xl | 48px | Not used in this phase |

Exceptions: Tab bar uses horizontal scrolling with `gap-0` between tabs (tabs are flush against each other, separated by 1px borders). XML tree indentation uses 16px (ml-4) per nesting level, consistent with the folder tree in Phase 2.

## Typography

| Role | Size | Weight | Line Height | Usage in Phase 3 |
|------|------|--------|-------------|-------------------|
| Body | 14px (text-sm) | 400 (normal) | 1.43 (20px) | Tab labels, file action labels, tree node element names, empty state body |
| Label | 12px (text-xs) | 400 (normal) | 1.33 (16px) | File size in header bar, attribute values in tree, Monaco line numbers |
| Heading | 14px (text-sm) | 600 (font-semibold) | 1.43 (20px) | Filename in content header bar |
| Display | 20px (text-xl) | 600 (font-semibold) | 1.4 (28px) | Empty state heading (no-file-open state) |

Two-weight contract: **400 regular** and **600 semibold**. No other weights.

Monaco Editor uses its own font stack: `ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, Liberation Mono, monospace` at 13px with 20px line height (matching existing `SqlCodeBlock` pattern but at 13px for full-file viewing).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` / oklch(1 0 0) light, oklch(0.145 0 0) dark | Content area background, Monaco editor background (via vs/vs-dark theme), XML tree view background |
| Secondary (30%) | `var(--muted)` / oklch(0.97 0 0) light, oklch(0.269 0 0) dark | Tab bar background, content header bar background, inactive tab background |
| Accent (10%) | `var(--accent)` / oklch(0.97 0 0) light, oklch(0.269 0 0) dark | Active tab indicator, hovered tab background, tree/source toggle active state |
| Destructive | `var(--destructive)` | Not used in Phase 3 (no destructive actions) |

Additional semantic colors for this phase:

| Role | Value | Usage |
|------|-------|-------|
| Active tab bottom border | `var(--primary)` / oklch(0.205 0 0) light, oklch(0.922 0 0) dark | 2px bottom border on the active tab |
| XML element tag | `var(--foreground)` | Element tag names in XML tree view |
| XML attribute name | `oklch(0.55 0.15 250)` light, `oklch(0.72 0.12 250)` dark (blue-ish, matching Monaco XML attribute highlighting) | Attribute names in XML tree (e.g., `id`, `type`) |
| XML attribute value | `oklch(0.55 0.12 150)` light, `oklch(0.72 0.10 150)` dark (green-ish, matching Monaco XML string highlighting) | Attribute values in XML tree (e.g., `"123"`, `"new"`) |
| XML comment | `var(--muted-foreground)` | Comment text and PI text in XML tree, italic style |
| XML CDATA | `oklch(0.55 0.08 60)` light, `oklch(0.72 0.06 60)` dark (warm muted tone) | CDATA section content in XML tree |
| Parse error banner | `oklch(0.75 0.15 85)` (amber-500 equivalent) | Warning banner background when XML tree parse fails |
| Muted text | `var(--muted-foreground)` | File size text, tab tooltip path, "No file open" body copy |

Accent reserved for: active tab bottom border indicator, hovered tab background (`hover:bg-accent`), tree/source segmented control active segment. Never used for text color or icons.

## Layout Specifications

### Explorer Shell (updated from Phase 2)

```
+--------------------------------------------------+
| ExplorerNavBar (h-auto, ~48px)                   |
+-------------+------------------------------------+
| Explorer    | FileTabBar (h-9, scrollable)       |
| Sidebar     +------------------------------------+
| (280px      | FileContentHeader (h-10)           |
| resizable   +------------------------------------+
| collapsible)|                                    |
|             | XmlSourceView or XmlTreeView       |
|             | (flex-1, scrollable)               |
|             |                                    |
+-------------+------------------------------------+
```

When no file is open, `FileTabBar` and `FileContentHeader` are not rendered. The full content area shows `ExplorerEmptyState`.

### FileTabBar

```
+--------------------------------------------------------------+
| [FileCode] file1.xml  [X] | [FileText] data.csv  [X] | ...  |
|  active (border-b-2)      |  inactive                 |      |
+--------------------------------------------------------------+
```

- Container: `flex items-center h-9 bg-muted border-b overflow-x-auto overflow-y-hidden`
- No scrollbar visible: `scrollbar-width: none; -webkit-scrollbar: display: none` (tabs scroll via mouse wheel or trackpad gesture)
- Individual tab: `flex items-center gap-1.5 px-4 h-full text-sm border-r border-border cursor-pointer whitespace-nowrap`
- Active tab: `bg-background border-b-2 border-b-primary` (background matches content area, 2px bottom accent border)
- Inactive tab: `bg-muted text-muted-foreground hover:bg-accent hover:text-accent-foreground`
- Tab icon: `h-3.5 w-3.5 flex-shrink-0` (`FileCode` for XML, `FileText` for non-XML)
- Tab label: `text-sm truncate max-w-[160px]` (truncates long filenames)
- Close button: `h-5 w-5 rounded-sm opacity-0 group-hover:opacity-100` (visible on tab hover), `hover:bg-muted-foreground/20`. Always visible on active tab.
- Tab tooltip: full file path, shown via `Tooltip` component on hover over the label area
- Duplicate name disambiguation: if two open files share the same filename, append ` - {parentFolder}` (e.g., `file1.xml - 20251223`)

### FileContentHeader

```
+--------------------------------------------------------------+
| [FileCode] filename.xml   12.4 KB  | [Tree|Source] | [actions]|
+--------------------------------------------------------------+
```

- Container: `flex items-center gap-2 px-4 h-10 border-b bg-muted/50`
- Left section: `flex items-center gap-2 min-w-0 flex-1`
  - File icon: `h-4 w-4 flex-shrink-0 text-muted-foreground`
  - Filename: `text-sm font-semibold truncate`
  - File size: `text-xs text-muted-foreground flex-shrink-0 ml-1`
- Center-right: Tree/Source segmented control
  - Container: `flex items-center h-7 rounded-md bg-muted border`
  - Each segment: `px-2.5 h-full flex items-center gap-1.5 text-xs rounded-sm cursor-pointer`
  - Active segment: `bg-background shadow-sm`
  - Inactive segment: `text-muted-foreground hover:text-foreground`
  - Tree icon: `TreePine` at `h-3.5 w-3.5`
  - Source icon: `Code` at `h-3.5 w-3.5`
  - Non-XML files: toggle hidden (only source view available)
  - XML parse failure: tree segment disabled (`opacity-50 cursor-not-allowed`), tooltip: "Unable to parse XML"
- Right section: `flex items-center gap-1`
  - `Separator` vertical, `h-5`
  - Copy path button: `variant="ghost" size="icon" className="h-7 w-7"` with `Copy` icon (h-3.5 w-3.5), tooltip: "Copy file path"
  - Copy content button: `variant="ghost" size="icon" className="h-7 w-7"` with `ClipboardCopy` icon (h-3.5 w-3.5), tooltip: "Copy raw content"

### XmlSourceView

- Container: `flex-1 overflow-hidden` (Monaco fills entire area)
- Monaco Editor configuration:
  - Language: `"xml"`

  - Read-only: `true`, `domReadOnly: true`

  - Line numbers: `"on"`
  - Folding: `true` (XML element folding enabled, per D-06)
  - Font size: `13`
  - Line height: `20`
  - Font family: `ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, Liberation Mono, monospace`
  - Word wrap: `"on"`
  - Scroll beyond last line: `false`
  - Automatic layout: `true`
  - Context menu: `false`
  - Render line highlight: `"line"` (subtle current-line highlight for orientation)
  - Glyph margin: `false`
  - Overview ruler: `0` lanes
  - Padding: `{ top: 8, bottom: 8 }`
  - Scrollbar: `{ alwaysConsumeMouseWheel: false }`
  - Aria label: `"File source view"`

### XmlTreeView

```
+--------------------------------------------------------------+
| v [FileCode] Root                                            |
|   v [FileCode] Patient  id="123"  type="new"                |
|     > [FileCode] Name                                        |
|     > [FileCode] Address                                     |
|       [Type] John Doe                                        |
|     [MessageSquare] <!-- Patient record -->                  |
|   > [FileCode] Insurance                                     |
+--------------------------------------------------------------+
```

- Container: `flex-1 overflow-auto p-2` with `ScrollArea`
- Tree starts fully collapsed: only root element visible (D-08)
- Each row: `XmlTreeNode`
- Row height: 28px (py-1 + content), consistent with folder tree nodes

### XmlTreeNode

- Row: `flex items-center gap-1 w-full rounded py-1 hover:bg-muted cursor-pointer`
- Indentation: `paddingLeft: {depth * 16}px` (same 16px per level as folder tree)
- Chevron: `h-4 w-4 text-muted-foreground flex-shrink-0` (`ChevronRight` / `ChevronDown`). Hidden for leaf nodes (text nodes, comments without children).

Node type visual specifications:

| Node Type | Icon | Name Style | Additional Content |
|-----------|------|------------|-------------------|
| Element | `FileCode` h-3.5 w-3.5 | `text-sm text-foreground` | Attributes inline: `text-xs` name in attribute-name color, `=` in muted, `"value"` in attribute-value color |
| Text | `Type` h-3.5 w-3.5 text-muted-foreground | `text-sm text-foreground truncate max-w-[400px]` | Text content displayed directly. Long text truncated with ellipsis. |
| Comment | `MessageSquare` h-3.5 w-3.5 text-muted-foreground | `text-sm text-muted-foreground italic truncate max-w-[400px]` | Shows `<!-- {content} -->` |
| Processing Instruction | `Hash` h-3.5 w-3.5 text-muted-foreground | `text-sm text-muted-foreground truncate max-w-[400px]` | Shows `<?{target} {data}?>` |
| CDATA | `Braces` h-3.5 w-3.5 text-muted-foreground | `text-sm truncate max-w-[400px]` in CDATA color | Shows `<![CDATA[{content}]]>` |

Attribute rendering within an element node row:
```
[ChevronRight] [FileCode] Patient  id="123"  type="new"
                          ^^^^^^^^ ^^^ ^^^^  ^^^^ ^^^^^
                          element  attr=val  attr  val
                          name     name val  name  val
```
- Attribute name: `text-xs` in XML attribute name color
- Equals sign: `text-xs text-muted-foreground`
- Attribute value: `text-xs` in XML attribute value color, wrapped in quotes
- Attributes separated by 8px gap (gap-2)
- Maximum 5 attributes shown inline; overflow shows `+N more` in `text-xs text-muted-foreground`

### Parse Error Banner

When DOMParser fails to parse the XML file (malformed):

```
+--------------------------------------------------------------+
| [AlertTriangle] Unable to parse XML tree -- showing source   |
+--------------------------------------------------------------+
```

- Container: `flex items-center gap-2 px-4 py-2 bg-amber-50 dark:bg-amber-950/30 border-b border-amber-200 dark:border-amber-800`
- Icon: `AlertTriangle` at `h-4 w-4 text-amber-600 dark:text-amber-500`
- Text: `text-sm text-amber-800 dark:text-amber-200`
- Position: immediately below the `FileContentHeader`, above the source view
- Behavior: auto-switches to source view when tree view cannot be rendered. Tree toggle disabled.

### File Loading State

When file content is being read from the filesystem:

- Tab shows filename with `Loader2` spinner replacing the file icon (h-3.5 w-3.5 animate-spin)
- Content area shows centered `Skeleton` placeholder: `h-4 w-3/4` and `h-4 w-1/2` and `h-4 w-2/3` stacked vertically with `gap-2`
- Header bar shows filename but file size shows `--` until loaded

## Interaction Contracts

### File Opening

| Trigger | Action | Result |
|---------|--------|--------|
| Click file node in sidebar tree | Read file content via `read_file_cmd` Rust command, add tab to store | New tab appears in `FileTabBar`, tab becomes active, content loads in source view (default per D-02) |
| Click file already open in a tab | Switch to existing tab (D-14) | Active tab changes, content area shows that file's view mode and scroll position |
| Click tab | Switch active tab | Content area shows that tab's content, view mode, and scroll position |

### Tab Management

| Trigger | Action | Result |
|---------|--------|--------|
| Click tab close button (X) | Remove tab from store | Tab removed. If active, adjacent tab becomes active (right neighbor preferred, then left). If last tab, show empty state. |
| Right-click tab | Show context menu | Menu items: Close, Close Others, Close All, separator, Copy Path, Open in External Editor, Copy Content, Save Copy |
| Middle-click tab | Close tab | Same as clicking close button |
| Scroll (mouse wheel on tab bar) | Scroll tabs horizontally | Tab bar scrolls left/right. No vertical scroll. |

### View Mode Toggle

| Trigger | Action | Result |
|---------|--------|--------|
| Click "Tree" segment | Switch to XML tree view | Content body renders `XmlTreeView`. Scroll position preserved per tab. |
| Click "Source" segment | Switch to source view | Content body renders `XmlSourceView` (Monaco). Scroll position preserved per tab. |
| Open non-XML file | Source view only | Tree/Source toggle hidden in header bar. Monaco renders with plain text or auto-detected language. |
| Open malformed XML | Source view forced | Parse error banner shown. Tree toggle disabled with tooltip. Source view renders the raw content. |

### XML Tree Expand/Collapse

| State | Visual | Interaction |
|-------|--------|-------------|
| Collapsed element | `ChevronRight` icon, no children visible | Click row or chevron to expand and show child nodes |
| Expanded element | `ChevronDown` icon, children rendered below | Click row or chevron to collapse |
| Leaf node (text, comment, PI, CDATA) | No chevron (16px spacer in chevron position) | No expand/collapse. Click selects row. |

### File Actions

| Action | Trigger | Behavior |
|--------|---------|----------|
| Copy path | Click `Copy` icon in header bar, or context menu | Copy full file path to system clipboard. Brief toast or visual confirmation. |
| Copy content | Click `ClipboardCopy` icon in header bar, or context menu | Copy entire raw file content to system clipboard. Brief toast. |
| Open in external editor | Context menu only | Invoke `tauri::api::shell::open` with file path. OS opens default application for file type (D-18). |
| Save copy | Context menu only | Open Tauri native save dialog (`@tauri-apps/plugin-dialog` `save()`). Write file to chosen location (D-19). |
| Close tab | Close button, context menu "Close", middle-click | Remove tab from store. |
| Close others | Context menu "Close Others" | Close all tabs except the right-clicked tab. That tab becomes active. |
| Close all | Context menu "Close All" | Close all tabs. Show empty state. |

### State Management

Each open tab maintains independently (per D-15, D-12):
- `id`: unique identifier (file path)
- `filePath`: full filesystem path
- `fileName`: display name (with disambiguator if needed)
- `content`: raw file content string (loaded once)
- `viewMode`: `"source"` or `"tree"` (default: `"source"` per D-02)
- `scrollPosition`: `{ source: number; tree: number }` (scroll top for each view)
- `isXml`: boolean (determines whether tree view toggle is available)
- `parseError`: boolean (true if DOMParser fails on XML content)
- `isLoading`: boolean (true while file read is in progress)

Tabs do NOT persist across sessions (D-13). App opens with no tabs and empty state.

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | All tabs reachable via Tab key. Active tab focused. Close button focusable within tab. Arrow left/right to switch between tabs. |
| Tab close | Tab close button has `aria-label="Close {filename}"` |
| View mode toggle | Segmented control uses `role="tablist"` with `role="tab"` on each segment. `aria-selected` on active segment. |
| XML tree navigation | Arrow up/down to move between tree rows. Arrow right to expand, arrow left to collapse. `role="tree"` on container, `role="treeitem"` on nodes. |
| File actions | Action buttons have tooltips that serve as accessible names via `aria-label`. |
| Screen reader | Tab bar announces active tab count: "{N} files open, viewing {filename}" via `aria-label` on tab bar container. |
| Focus indicators | Default shadcn focus-visible ring: `focus-visible:ring-1 focus-visible:ring-ring` |
| Color contrast | All text uses `--foreground` on `--background` or `--muted-foreground` on `--background`. XML tree semantic colors (attribute name/value) maintain WCAG AA contrast against `--background`. |
| Monaco accessibility | `ariaLabel: "File source view"` on Monaco editor. Monaco has built-in screen reader support. |
