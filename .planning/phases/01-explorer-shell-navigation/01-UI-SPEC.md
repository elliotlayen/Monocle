## Typography

| Role | Size | Weight | Line Height | Usage in Phase 1 |
|------|------|--------|-------------|-------------------|
| Heading | 20px (`text-xl`) | 600 (semibold) | 1.2 (24px) | Empty state feature name heading |
| Body | 14px (`text-sm`) | 400 (regular) | 1.43 (20px) | Button labels, nav bar text, empty state body copy, empty state description paragraph |
| Label | 12px (`text-xs`) | 400 (regular) | 1.33 (16px) | Keyboard shortcut badges, status bar text |

Two-weight contract: **400 regular** and **600 semibold**. No other weights permitted.

- 400 regular -- Label (12px), Body (14px), description paragraph (14px with `text-muted-foreground` for visual distinction)
- 600 semibold -- Heading (20px), branding override (16px)

Branding override: "Monocle" text in nav bar uses JetBrains Mono at 16px (`text-base`) weight 600 (`font-semibold`). This matches the existing toolbar pattern exactly.

## Color

Phase 1 uses the existing app color system with no additions. All colors reference CSS custom properties defined in `src/index.css` using oklch.

| Role | Token | Light Value | Dark Value | Usage |

| Dominant (60%) | `--background` | `oklch(1 0 0)` | `oklch(0.145 0 0)` | Explorer content area background |

Accent reserved for: button hover background (`hover:bg-accent`). No other accent usage in Phase 1.

## Interaction Contracts

### Mode Entry

| Trigger | Action | Result |
|---------|--------|--------|
| Click "Integration Explorer" on home screen | Call `onEnterExplorer` callback | Render `ExplorerShell` instead of `HomeScreen` |
| Press `Cmd+E` / `Ctrl+E` from home screen | Same as click | Same as click |

### Mode Exit

| Trigger | Action | Result |
|---------|--------|--------|
| Click Home button in nav bar | Return to home screen immediately | Render `HomeScreen`. No confirmation dialog (D-04: Explorer is read-only). |

### Settings Access

| Trigger | Action | Result |
|---------|--------|--------|
| Click Settings gear in nav bar | Open `AppSettingsSheet` | Same settings sheet used by other modes |
| Click "Open Settings" CTA in empty state | Open `AppSettingsSheet` | Same behavior |

### Keyboard Shortcuts

| Shortcut | Context | Action |
|----------|---------|--------|
| `Cmd+E` / `Ctrl+E` | Home screen | Enter Explorer mode |
| `Cmd+,` / `Ctrl+,` | Explorer mode | Open Settings (existing app-wide shortcut) |

### State Preservation

- Entering Explorer mode does NOT affect Schema Graph state (schema data, filters, focus remain intact)
- Exiting Explorer mode returns to home screen; user can then connect to a server or enter canvas mode normally
- Mode state: new `"explorer"` value added to the existing `mode` field in `useSchemaStore`, or a new top-level state in `App.tsx`

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | All interactive elements reachable via Tab. Buttons use native `<button>` via shadcn `Button`. |
| Focus indicators | Default shadcn focus-visible ring: `focus-visible:ring-1 focus-visible:ring-ring` |
| Screen reader labels | Tooltip text serves as accessible name for icon-only nav bar buttons. Home screen button has visible text label. |
| Color contrast | All text uses `--foreground` on `--background` or `--muted-foreground` on `--background`. oklch neutral palette ensures WCAG AA. |
| Keyboard shortcut | `Cmd+E` / `Ctrl+E` registered at app level in `App.tsx` `useEffect` handler, matching existing `Cmd+K` pattern. |
