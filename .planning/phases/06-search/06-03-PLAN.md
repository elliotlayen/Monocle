    CREATE use-search-highlight.ts (per D-22, mirrors use-validation-decorations.ts):
    Export `useSearchHighlight` function. Parameters: `editorInstance: editor.IStandaloneCodeEditor | null`, `searchTerms: string[] | null`. Import `useRef`, `useEffect` from react, `editor` type from `monaco-editor`.

    Implementation following the decoration hook pattern from use-validation-decorations.ts:
    - Maintain a `collectionRef = useRef&lt;editor.IEditorDecorationsCollection | null&gt;(null)`
    - useEffect depending on `[editorInstance, searchTerms]`:
      - If no editorInstance or no searchTerms or empty searchTerms array: clear existing collection via `collectionRef.current?.clear()`, return.
      - Get model via `editorInstance.getModel()`. If no model, return.
      - Build decorations array: for each term in searchTerms, call `model.findMatches(term, false, false, false, null, false)` (searchString, searchOnlyEditableRange=false, isRegex=false, matchCase=false, wordSeparators=null, captureMatches=false). Track the first match across all terms for scroll targeting.
      - Map each FindMatch result to `IModelDeltaDecoration` with:
        - `range: match.range`
        - `options.className: "search-match-highlight"` (CSS class for background highlight)
        - `options.overviewRuler: { color: "oklch(0.82 0.12 85)", position: 2 }` (OverviewRulerLane.Center -- amber markers in minimap per UI-SPEC)
      - For the very first match (across all terms), additionally use a brighter highlight: `options.className: "search-match-highlight-current"` to indicate the scrolled-to match.
      - Create collection: `collectionRef.current = editorInstance.createDecorationsCollection(decorations)`
      - Scroll to first match: `editorInstance.revealRangeInCenterIfOutsideViewport(firstMatch.range)` (per D-22)
      - Cleanup in useEffect return: clear collection.

    Add a CSS injection (either in the hook via Monaco's defineTheme or as a style tag) for the two CSS classes. Use the colors from UI-SPEC:
    - `.search-match-highlight`: `background: oklch(0.82 0.12 85 / 0.25)` light, `oklch(0.80 0.12 85 / 0.20)` dark. Since Monaco does not use CSS custom properties for decoration classes, inject styles via a `<style>` element or use `monaco.editor.defineTheme` extension.
