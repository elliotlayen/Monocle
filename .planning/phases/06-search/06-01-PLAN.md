Parameters: `app: AppHandle`, `query: String`, `folder_paths: String` (JSON array of folder path strings -- supports single folder and "All sources" per D-09, RESEARCH assumption A4), `file_pattern: String`, `scope_label: String`, `operation_id: String`, `explorer_state: State&lt;'_, ExplorerState&gt;`. Implementation:

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend -> Rust IPC | Search query and folder paths cross from untrusted frontend input to filesystem operations |
| Filesystem -> Rust | File content read from potentially hostile network shares crosses into string processing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | content_search_cmd folder_paths | mitigate | Deserialize folder_paths JSON; validate each path exists via std::fs::metadata before walking. Do not allow paths outside configured source roots. |
| T-06-02 | Denial of Service | content_search_cmd large scan | mitigate | CancellationToken checked per file iteration; event throttling at 50ms prevents frontend flooding |
| T-06-03 | Denial of Service | parse_search_terms | accept | Query is a short user-typed string; no regex involved (D-05/D-06). Simple string ops on bounded input. |
| T-06-04 | Information Disclosure | search-result events | accept | Events contain file paths and match counts only (D-29), no file content. Desktop app, no network exposure. |
| T-06-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this phase. All dependencies already audited and present in lockfiles. |
