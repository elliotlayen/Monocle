## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps in problem rows, inline padding, gutter icon padding |
| sm | 8px | Problem row internal padding, status bar element gaps, panel header gaps |
| lg | 16px | Problems panel horizontal padding (`px-4`), status bar horizontal padding (`px-4`) |
| xl | 24px | Section spacing (not used in this phase) |
| 2xl | 32px | Not used in this phase |

Exceptions: Problems panel default height is 200px, resizable via drag handle, min 100px, max 50% of the content area height. Status bar height is 24px (`h-6`). Drag handle hit area is 4px, consistent with the sidebar resize handle pattern.

### ProblemsPanel

```
+--------------------------------------------------------------+
| [v] Problems (3)          [2 errors]  [1 warning]            |
+--------------------------------------------------------------+
| [CircleAlert]  1:15   Null byte (0x00) detected              |
| [CircleAlert]  5:28   Unescaped ampersand in content          |
| [TriangleAlert] 1:1   BOM detected (UTF-8 byte order mark)   |
+--------------------------------------------------------------+
```

- Outer container: `flex flex-col border-t` with dynamic height from drag state
- Header: `flex items-center gap-2 px-4 h-8 bg-muted/50 cursor-pointer border-b` (clickable to toggle collapse)
  - Chevron: `h-3.5 w-3.5 text-muted-foreground flex-shrink-0` (`ChevronDown` when expanded, `ChevronRight` when collapsed)
  - Title: `text-sm font-semibold` showing `"Problems ({N})"` where N is total count
  - Spacer: `flex-1`
  - Error count indicator: `flex items-center gap-1` -- `CircleDot` icon in error color (h-3 w-3) + count `text-xs` in error color
  - Warning count indicator: `flex items-center gap-1` -- `AlertTriangle` icon in warning color (h-3 w-3) + count `text-xs` in warning color
  - Error/warning indicators only shown when respective count > 0
- Body: `ScrollArea` wrapping the problem list, `flex-1 overflow-hidden`
- When collapsed: only the header row is visible (h-8). Body hidden.

### ProblemRow

```
[CircleAlert]  5:28   Unescaped ampersand in content
```

- Row: `flex items-center gap-2 px-4 py-1.5 hover:bg-accent cursor-pointer text-sm`
- Severity icon: `h-4 w-4 flex-shrink-0` -- `CircleAlert` in error color or `TriangleAlert` in warning color
- Location: `text-xs text-muted-foreground font-mono flex-shrink-0 w-14 text-right` showing `{line}:{col}`
- Description: `text-sm truncate` showing the problem description text
- Click action: auto-switches to source view (if currently in tree view) and scrolls Monaco to the line/column. Sets cursor position.

### ValidationStatusBar

```
+--------------------------------------------------------------+
| [CircleDot] 2 errors  [AlertTriangle] 1 warning    UTF-8    |
+--------------------------------------------------------------+
```

- Container: `flex items-center gap-3 px-4 h-6 border-t bg-muted/30 text-xs cursor-pointer hover:bg-accent/50`
- Always visible when a file is open (even if no problems -- shows "No problems" + encoding)
- Left section: `flex items-center gap-3`
  - Error count (if > 0): `flex items-center gap-1` -- `CircleDot` icon (h-3 w-3, error color) + `"{N} error{s}"` text (text-xs, error color)
  - Warning count (if > 0): `flex items-center gap-1` -- `AlertTriangle` icon (h-3 w-3, warning color) + `"{N} warning{s}"` text (text-xs, warning color)
  - No problems: `"No problems"` text (text-xs, muted-foreground)
- Spacer: `flex-1`
- Right section: `flex items-center gap-1`
  - Encoding label: `text-xs text-muted-foreground` showing encoding name (e.g., `"UTF-8"`, `"Windows-1252"`, `"ISO-8859-1"`)
- Click anywhere on bar: toggles the problems panel open/closed

### Monaco Decorations (XmlSourceView changes)

Phase 4 modifies the existing `XmlSourceView` Monaco configuration:

**Changed options:**
- `glyphMargin`: `false` -> `true` (enables the gutter icon area left of line numbers)
- `overviewRulerLanes`: `0` -> `2` (enables the scrollbar-side markers)

**Decoration specifications:**

Gutter markers:
- Error: red circle icon in the glyph margin area. Uses `glyphMarginClassName` with a CSS class that renders a 14px red filled circle via `::before` pseudo-element.
- Warning: yellow circle icon in the glyph margin area. Same pattern, yellow fill.
- If a line has both errors and warnings, error takes precedence (red shown).

Inline highlights:
- Error character: `className` with semi-transparent red background (`bg-red-500/12 dark:bg-red-400/15`) and red bottom border (`border-b-2 border-red-500 dark:border-red-400`).
- Warning character: `className` with semi-transparent yellow background (`bg-amber-500/12 dark:bg-amber-400/15`) and yellow bottom border (`border-b-2 border-amber-500 dark:border-amber-400`).
- Decoration range: exact start column to end column for the problem character/sequence. For single-character problems (null byte, control char), the range is 1 character wide.

Overview ruler markers:
- Error: red marker in the overview ruler (right-side scrollbar track). Uses `overviewRuler: { color: 'oklch(0.577 0.245 27.325)', position: 4 }` (Monaco `OverviewRulerLane.Right`).
- Warning: yellow marker. Uses `overviewRuler: { color: 'oklch(0.75 0.15 85)', position: 4 }`.

Hover messages:
- Each decoration includes `hoverMessage: { value: '{problem description}' }` so hovering over a highlighted character in Monaco shows the problem text.

### Validation Dot Badges on Tree File Nodes

Added to `FolderTreeNode` after the filename and before any existing badges:

```
[FileCode] filename.xml [red dot]
[FileCode] clean-file.xml
[FileCode] bom-file.xml [yellow dot]
```

- Error badge: `h-2 w-2 rounded-full flex-shrink-0` with `bg-red-500 dark:bg-red-400` (solid red dot, 8px diameter)
- Warning badge: `h-2 w-2 rounded-full flex-shrink-0` with `bg-amber-500 dark:bg-amber-400` (solid yellow dot, 8px diameter)
- No badge: file is either clean or not yet validated. No visual indicator.
- Badge placement: between the filename text and the child count or star icon. Uses `ml-1` gap from filename.

## Interaction Contracts

### Validation on File Open

| Trigger | Action | Result |
|---------|--------|--------|
| File opened via sidebar click or tab switch to uncached file | `read_file_cmd` returns content + problems + encoding + hasBom | Validation results stored in tab state and validation cache. Monaco decorations applied. Status bar updated. |
| File already validated (in cache) | Skip re-validation, load cached results | Same visual state as fresh validation. No IPC call for validation. |
| File has errors or warnings | Problems panel auto-shows if currently hidden | Panel slides open. Status bar reflects counts. Tree badge applied. |
| File is clean (no problems) | Problems panel stays hidden (or stays in user's last state) | Status bar shows "No problems" + encoding. No badge on tree node. |

### Problems Panel Behavior (D-01, D-02, D-03)

| State | Visual | Interaction |
|-------|--------|-------------|
| Hidden (no problems, never shown) | Not rendered. Only status bar visible. | Click status bar to manually open (shows empty panel with "No problems detected"). |
| Auto-shown (file opened with problems) | Panel appears below source/tree view at 200px default height | Drag handle to resize. Click header chevron to collapse. |
| Collapsed | Only panel header visible (h-8), body hidden | Click header to expand. Panel remembers last height. |
| Expanded | Header + scrollable problem list | Click header to collapse. Drag handle to resize. |
| Tab switch | Panel updates to show new active tab's problems | If new tab has problems and panel was collapsed by user, panel stays collapsed. If new tab has problems and panel was auto-hidden (clean file), panel auto-shows (D-02). |

### Problems Panel Resize

| State | Visual | Interaction |
|-------|--------|-------------|
| Default | 200px panel height | Drag handle at top of panel |
| Resizing | Panel height follows mouse, source/tree view shrinks proportionally | Min 100px, max 50% of content area height |
| Drag handle | 4px transparent hit area at top border of panel, `cursor-row-resize` on hover | Mouse down starts drag, mouse move resizes, mouse up ends drag |

### Problem Row Click (D-21)

| Trigger | Action | Result |
|---------|--------|--------|
| Click problem row (currently in source view) | Monaco scrolls to line and sets cursor at column | Line is visible and cursor blinks at the problem location |
| Click problem row (currently in tree view) | Auto-switch to source view, then scroll and set cursor | View mode changes to source, Monaco loads and jumps to problem location |
| Click problem row (non-XML file) | Scroll to line and set cursor | No view mode switch needed (always source) |

### Status Bar Click (D-04)

| Trigger | Action | Result |
|---------|--------|--------|
| Click anywhere on status bar | Toggle problems panel visibility | If panel hidden/collapsed, expand it. If panel expanded, collapse it. |

### Validation Cache (D-15, D-16)

| Scenario | Behavior |
|----------|----------|
| File opened first time | Validation runs as part of `read_file_cmd`. Results cached by file path in store. |
| File tab closed, then re-opened | Cached validation results used. No re-validation IPC call. |
| File path re-opened from different location in tree | Same path = same cache entry. Reuses cached results. |
| App restart | Cache cleared. Files re-validated on next open. |
| Session scope | Cache persists for entire app session only (D-15). |

### Tree Badge Persistence (D-14, D-15, D-17, D-18)

| Scenario | Behavior |
|----------|----------|
| File opened and validated with errors | Red dot badge appears on file node. Persists even after closing the tab. |
| File opened and validated with warnings only | Yellow dot badge appears. Persists after closing tab. |
| File opened and validated as clean | No badge. No badge = not validated or clean (D-17). |
| App restart | All badges cleared. Only re-appear when files are opened again. |
| Parent folders | No aggregate badges (D-18). Phase 5 handles folder-level indicators. |

## State Management Extensions

### FileTab Type Extensions

New fields added to the existing `FileTab` interface:

| Field | Type | Description |
|-------|------|-------------|
| `problems` | `ValidationProblem[]` | Array of validation problems from Rust backend |
| `encoding` | `string` | Detected file encoding (e.g., "UTF-8", "Windows-1252") |
| `hasBom` | `boolean` | Whether a BOM was detected |

### New Types

```
interface ValidationProblem {
  line: number;
  column: number;
  endColumn: number;
  message: string;
  severity: "error" | "warning";
  code: string; // e.g., "null-byte", "unescaped-ampersand", "bom-detected"
}

type ValidationStatus = "error" | "warning" | "clean";
```

### Explorer Store Extensions

New state:

| Field | Type | Description |
|-------|------|-------------|
| `validationCache` | `Map<string, { problems: ValidationProblem[], encoding: string, hasBom: boolean }>` | Session-scoped cache keyed by file path |
| `problemsPanelOpen` | `boolean` | Whether the problems panel is expanded |
| `problemsPanelHeight` | `number` | Current panel height in pixels (default 200) |

New actions:

| Action | Description |
|--------|-------------|
| `toggleProblemsPanel` | Toggle panel open/collapsed state |
| `setProblemsPanelHeight` | Update panel height from drag resize |
| `getValidationStatus(filePath: string)` | Returns `"error"` / `"warning"` / `"clean"` / `undefined` from cache |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Problems panel keyboard | Panel header focusable, Enter/Space toggles collapse. Problem rows focusable via Tab, Enter clicks to jump. Arrow up/down to navigate rows. |
| Panel ARIA | Container: `role="region" aria-label="Problems panel"`. Header: `aria-expanded="{true/false}"`. List: `role="list"`. Rows: `role="listitem"`. |
| Status bar | `role="status" aria-live="polite"` so screen readers announce count changes when switching tabs. `aria-label="Validation status: {N} errors, {N} warnings, encoding {name}"`. |
| Gutter markers | Monaco's built-in `hoverMessage` provides accessible hover text. Glyph margin decorations are visual-only; the problems panel is the accessible path to the same information. |
| Color contrast | Error red and warning yellow both maintain WCAG AA contrast against `--background` for icon usage. Inline highlight backgrounds are intentionally low-contrast (decorative overlay), with the problem information accessible via panel and hover. |
| Focus indicators | Default shadcn focus-visible ring: `focus-visible:ring-1 focus-visible:ring-ring` on panel header, problem rows, and status bar. |
| Tree badges | Dot badges are supplemental visual indicators (8px diameter). The badge presence is not conveyed via `aria-label` on the tree node -- the validation status is accessible through the problems panel and status bar when the file is open. |
